# 基于OpenCV的运动检测远程监控系统（Qt客户端）

本项目是运动检测远程监控系统的Qt客户端，负责接收树莓派服务端的视频流并实时显示，支持远程调节灵敏度和手动抓拍。

## 功能特点

- **实时视频显示**：通过TCP接收服务端发送的JPEG流，解码后显示在窗口中，支持窗口大小自适应。
- **灵敏度调节**：滑动滑块实时发送 `set_area <阈值>` 命令，远程调整运动检测的面积阈值。
- **手动抓拍**：点击抓拍按钮，弹出文件保存对话框，用户可选择路径保存当前画面（JPEG/PNG格式）。
- **日志记录**：显示连接状态、命令发送、抓拍结果、错误信息等，便于调试。
- **丢帧策略**：限制显示帧率（最高10fps），避免界面卡顿，但抓拍仍使用最新帧。
- **健壮网络处理**：自动重连提示、错误恢复、连接状态指示。

## 系统要求

- **操作系统**：Windows 7/10/11 或 Linux（Ubuntu/Debian等）
- **Qt版本**：5.15 或更高（需包含Network和Widgets模块）
- **开发环境**：Qt Creator（推荐）

## 安装与运行

### 1. 安装Qt

从 [Qt官网](https://www.qt.io/download) 下载开源版Qt，安装时选择 `Qt 5.15.x` 以及 `Qt Creator`。

### 2. 获取源码

将本仓库代码克隆到本地：
```bash
git clone https://github.com/yourname/MotionMonitor.git
```

### 3. 使用Qt Creator打开项目

- 启动Qt Creator，点击“打开项目”，选择项目根目录下的 `MotionMonitor.pro` 文件。
- 选择构建套件（如 Desktop Qt 5.15.2 MinGW 64-bit），点击“配置项目”。

### 4. 编译与运行

- 点击左下角绿色三角形（运行）按钮，程序编译并启动。
- 界面出现后，输入树莓派服务端的IP地址和端口（默认视频端口8888，命令端口8889）。
- 点击“连接”按钮，若服务端已运行，视频画面将开始显示。

## 使用说明

### 连接树莓派

1. 确保树莓派上的服务端程序已启动（见服务端README）。
2. 在客户端界面输入树莓派IP和端口（视频端口8888，命令端口8889）。
3. 点击“连接”按钮，状态栏显示“已连接成功”，视频开始播放。

### 调节灵敏度

- 拖动“灵敏度”滑块（范围0~2000），客户端会立即发送 `set_area <值>` 命令到服务端。
- 服务端根据新阈值筛选运动区域，数值越小越灵敏（小物体也会触发），数值越大只检测较大运动。

### 手动抓拍

- 点击“抓拍”按钮，客户端会弹出文件保存对话框，默认文件名包含时间戳（如 `snapshot_20260323_143045_123.jpg`）。
- 选择保存路径和文件名（可改为PNG格式），点击“保存”，画面即保存到指定位置。
- 日志窗口会显示保存成功或失败信息。

### 查看日志

- 日志窗口（下方文本框）记录所有操作和状态信息，包括连接、断开、命令发送、抓拍结果等。

## 网络协议

- **视频流端口**：8888  
  接收格式：4字节长度（网络字节序，uint32_t） + JPEG图像数据。
- **命令端口**：8889  
  发送文本命令，每行一个命令：
  - `set_area <area>`：设置运动面积阈值（单位：像素）。
  - `snapshot`：发送抓拍请求（服务端可扩展保存功能）。

## 常见问题及解决方案

1. **连接失败**
   - 检查树莓派IP是否正确，是否与电脑在同一局域网。
   - 确认服务端已运行（在树莓派执行 `netstat -tlnp | grep 8888`）。
   - 关闭Windows防火墙或允许程序访问网络。

2. **视频画面卡顿或延迟**
   - 降低服务端分辨率/帧率（如 `--width 320 --height 240 --framerate 5`）。
   - 客户端已实现丢帧策略（最高10fps），若仍卡顿可调整服务端JPEG质量（`cv::imencode` 中的质量参数）。

3. **抓拍保存失败**
   - 确保保存路径有写入权限（例如桌面或我的文档）。
   - 检查磁盘空间是否充足。
   - 日志窗口会显示具体错误信息。

4. **图像解码失败**
   - 可能是网络传输丢包导致帧数据损坏，服务端会持续发送，客户端自动丢弃损坏帧。
   - 若频繁出现，检查网络稳定性或降低视频质量。
